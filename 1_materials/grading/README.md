# Grading Scripts and Evaluation

This subfolder contains the scripts, dependencies, and datasets specifically dedicated to evaluating the submitted work of the students and teams involved in the experiment.

## Directory Elements

- **`1_en_student_evaluation.py`**: A Python script designed to parse, analyze, and generate individual evaluation scores or feedback for each student.
- **`2_en_team_evaluation.py`**: A Python script used to evaluate the work produced by student teams collectively.
- **`3_en_global_evaluation.py`**: A script that consolidates evaluation data into a global picture, merging outputs into a final evaluation format.
- **`Experiment_data_files/`**: Contains raw data files and inputs required for the evaluation scripts.
- **`en_data_teams.csv`**: A CSV file mapping teams or providing data related to the respective student teams.
- **`students_profile_anonymized.csv`**: Anonymized profiles or mapping of students participating in the evaluation.
- **`requirements.txt`**: The required Python packages specific to these evaluation scripts (such as `pandas` and `requests`).
- **`results/`**: The output directory containing evaluation metric reports and finalized evaluation data generated by the scripts.

## Setup and Usage

Follow these steps to set up the environment and run the evaluation pipeline:

### Prerequisites

- Python 3.10 or higher is installed.

### Installation & Execution

1. Navigate to the `grading` directory:
   ```bash
   cd refactoring-swarm-study/1_materials/grading
   ```

2. Install the required Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Run the evaluation scripts chronologically to reproduce the grading logic:
   - First, evaluate individual students:
     ```bash
     python 1_en_student_evaluation.py
     ```
   - Second, evaluate the teams:
     ```bash
     python 2_en_team_evaluation.py
     ```
   - Finally, generate the global evaluation summary:
     ```bash
     python 3_en_global_evaluation.py
     ```

The final tabulated scores and outputs will be located in the `results/` folder or exported as new CSV files within the `grading` directory structure.
